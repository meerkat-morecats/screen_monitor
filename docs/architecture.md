# 项目架构规划

## 1. 产品目标

这是一个强提醒型休息桌面应用，不只是普通通知工具。它的核心是“定时进入休息状态”，并在关键时刻用全屏遮罩和倒计时打断当前工作流。

## 2. 总体架构

### 前端

前端负责：
- 设置页与可视化配置
- 当前工作状态展示
- 倒计时遮罩界面
- 用户触发的测试与预览
- 与 Rust 后端同步状态

### Rust / Tauri

Rust 负责：
- 精准定时调度
- 应用生命周期管理
- 多窗口创建与切换
- 系统级能力对接
- 配置持久化
- 后续的托盘、开机启动、全局快捷键扩展

## 3. 推荐模块边界

### `features/schedule`
负责专注时长、休息时长、提醒周期、每日启用时间段等配置。

### `features/settings`
负责声音、音量、全屏模式、遮罩文案、是否允许跳过等偏好项。

### `features/overlay`
负责遮罩层 UI、视觉倒计时、休息中状态表达。

### `features/timer`
负责当前阶段倒计时、下次提醒时间、进度展示。

### `services/tauri`
负责与 Rust 命令、事件和窗口能力打交道，避免业务组件直接依赖 Tauri API。

### `services/audio`
负责音频资源清单、试听、默认音效映射。

## 4. 状态流建议

推荐将状态拆成三层：

1. 配置状态
2. 运行时状态
3. UI 临时状态

### 配置状态

包括：
- 专注时长
- 休息时长
- 提示音
- 音量
- 遮罩文案
- 是否开机自动启动

这类数据最终应该持久化到本地。

### 运行时状态

包括：
- 当前阶段：`focus | break | paused`
- 当前阶段开始时间
- 当前阶段结束时间
- 下一个提醒时间
- 遮罩窗口是否可见

这类状态更适合由 Rust 持有，并同步到前端。

### UI 临时状态

包括：
- 设置面板是否展开
- 试听中音频
- 表单编辑态
- 预览态遮罩是否展示

## 5. 窗口规划

第一版建议至少保留两类窗口：

1. 主窗口
2. 休息遮罩窗口

### 主窗口

用于配置和查看当前状态。

### 休息遮罩窗口

用于全屏提示休息，建议特性：
- 独立窗口
- 置顶
- 全屏
- 简洁高对比倒计时
- 根据配置控制是否允许关闭或跳过

## 6. 事件通信建议

Rust -> Frontend：
- 调度状态变化
- 倒计时 tick
- 休息开始
- 休息结束

Frontend -> Rust：
- 保存配置
- 手动开始测试提醒
- 打开 / 关闭遮罩
- 播放测试音效

## 7. 初始化阶段实现策略

### 阶段 A
- 先把 UI 壳层和业务目录搭好
- 预留 Tauri 能力接口
- 完成可视化原型

### 阶段 B
- 接入 Tauri
- 打通配置存储
- 打通遮罩窗口

### 阶段 C
- 实现真正的后台调度和系统托盘
- 加入开机启动、快捷键、通知等增强能力
